# JWT Secret Utility

JWT Secret Utility is a small Node.js project that helps beginner developers generate, validate, and safely load a JWT secret key. It uses Node.js built-in security features and only one dependency: `dotenv` for reading `.env` files.

A JWT secret is a long, random string used by a backend application to sign and verify authentication tokens. If this secret is weak or leaked, attackers may be able to create fake tokens. This utility creates a 64-byte random secret, which becomes a 128-character hexadecimal string.

## Requirements

- Node.js installed.
- A terminal such as PowerShell, Command Prompt, Git Bash, or the VS Code terminal.
- Basic Git knowledge if you want to push the project to GitHub.

## Installation

Open the project folder:

```bash
cd jwt-secret-utility
```

Install dependencies:

```bash
npm install
```

This installs only:

```text
dotenv
```

## Generate a JWT Secret

Run:

```bash
node generate-secret.js
```

Or use the npm script:

```bash
npm run generate
```

The script uses:

```javascript
crypto.randomBytes(64).toString('hex')
```

`crypto.randomBytes(64)` asks Node.js to create 64 cryptographically secure random bytes. `.toString('hex')` converts those bytes into a readable string using hexadecimal characters. The final secret is 128 characters long.

The script prints the secret in RGB green and writes it to `.env` as:

```env
JWT_SECRET="your_generated_secret_here"
```

## Verify the JWT Secret

Run:

```bash
node verify-secret.js
```

Or:

```bash
npm run verify
```

The verifier checks that `JWT_SECRET`:

- Exists.
- Is a string.
- Is at least 32 characters long.
- Uses hexadecimal characters when generated by this tool.
- Matches the recommended 128-character generated format.

If the secret is missing or weak, the script prints an RGB red error message and exits with code `1`.

## Demo: Load the Secret

Run:

```bash
node app.js
```

Or:

```bash
npm run demo
```

The demo imports `load-secret.js`, validates the secret, and prints a success message without showing the secret value.

## Use in a Larger Application

Import the loader:

```javascript
import { jwtSecret } from './load-secret.js';
```

Then pass `jwtSecret` to your JWT signing library in your real application. This project does not install `jsonwebtoken` because the assignment allows only Node.js built-in modules plus `dotenv`.

Example shape in a larger app:

```javascript
import { jwtSecret } from './load-secret.js';

// Example only:
// const token = jwt.sign({ userId: user.id }, jwtSecret, { expiresIn: '1d' });
```

## Security Notes

Never commit `.env` to GitHub. This project includes `.env` in `.gitignore`:

```text
node_modules
.env
```

If your JWT secret is leaked, generate a new one immediately and make users log in again.

## RGB Console Color Requirement

All console colors in this project use ANSI RGB escape sequences. The code uses numeric red, green, and blue values, such as:

```javascript
'\x1b[38;2;0;255;0m'
'\x1b[38;2;255;0;0m'
'\x1b[38;2;100;149;237m'
```

The project does not use hex color values or named color strings for console output.

## File Overview

- `generate-secret.js` creates a secure 64-byte JWT secret and writes it to `.env`.
- `verify-secret.js` checks whether the current `JWT_SECRET` is present and safe.
- `load-secret.js` exports the validated secret for other Node.js files.
- `app.js` demonstrates that the loader works.
- `utils/colorLog.js` keeps all RGB ANSI console output in one place.
- `utils/secretValidation.js` keeps secret validation rules consistent.

## Troubleshooting

If `verify-secret.js` says `JWT_SECRET` is missing, run:

```bash
node generate-secret.js
```

If `load-secret.js` throws an error, check that `.env` exists in the project root and contains:

```env
JWT_SECRET="your_generated_secret_here"
```

If the terminal colors look strange, use a modern terminal such as PowerShell, Windows Terminal, Git Bash, or the VS Code terminal.
