# JWT Secret Utility

JWT Secret Utility is a small Node.js project that helps beginner developers generate, validate, and safely load a JWT secret key. It uses Node.js built-in security features and only one dependency: `dotenv` for reading `.env` files.

A JWT secret is a long, random string used by a backend application to sign and verify authentication tokens. If this secret is weak or leaked, attackers may be able to create fake tokens. This utility creates a 64-byte random secret, which becomes a 128-character hexadecimal string.

## Requirements

- Node.js installed.
- A terminal such as PowerShell, Command Prompt, Git Bash, or the VS Code terminal.
- Basic Git knowledge if you want to push the project to GitHub.

## Installation

Open the project folder:

```bash
cd jwt-secret-utility
```

Install dependencies:

```bash
npm install
```

This installs only:

```text
dotenv
```

## Generate a JWT Secret

Run:

```bash
node generate-secret.js
```

Or use the npm script:

```bash
npm run generate
```

The script uses:

```javascript
crypto.randomBytes(64).toString('hex')
```

`crypto.randomBytes(64)` asks Node.js to create 64 cryptographically secure random bytes. `.toString('hex')` converts those bytes into a readable string using hexadecimal characters. The final secret is 128 characters long.

The script prints the secret in RGB green and writes it to `.env` as:

```env
JWT_SECRET="your_generated_secret_here"
```

## Verify the JWT Secret

Run:

```bash
node verify-secret.js
```

Or:

```bash
npm run verify
```

The verifier checks that `JWT_SECRET`:

- Exists.
- Is a string.
- Is at least 32 characters long.
- Uses hexadecimal characters when generated by this tool.
- Matches the recommended 128-character generated format.
